# Web 前端优化与改进方案 — TypeScript + React

> 目标：将现有 PyQt5 桌面 GUI（[traffic_sign_system/ui](traffic_sign_system/ui/)）改造为可在浏览器中运行的 Web 前端，
> 同时复用 Python 后端的 `Predictor` / `VideoRecognizer` / `CameraRecognizer`。
> 本文档给出**架构选型、模块拆分、性能优化、部署与迁移**的完整方案。

---

## 1. 现状盘点

| 模块（PyQt） | 功能 | 改造方向 |
| --- | --- | --- |
| `main_window.py` | 三栏布局（控件 / 图像 / 结果） | React + Ant Design / shadcn/ui |
| `dashboard.py` | Matplotlib 三联图（置信度 / 混淆矩阵 / 历史） | Recharts + react-konva |
| `workers.py` | 4 个 QThread：LoadModel / Predict / Detect / Batch | Web Worker + 后端异步任务 |
| `predictor.py` | LRU 缓存、批量预测、缓存统计 | 后端 FastAPI 暴露同名接口 |
| `video_recognizer.py` | 帧间跳过 + ROI | 浏览器端 `MediaStream` → WebSocket → 后端推流 |
| `camera_recognizer.py` | 中心 ROI + 置信度平滑 | 同上，前端做 EMA 平滑 |
| `sign_detector.py` | HSV + 轮廓候选区域 | 后端推理；前端 Canvas 叠加 bbox |

桌面 GUI 已经把职责分得很干净，迁移到 Web 时**不要重写业务逻辑**，只把"渲染 + 用户交互"换成 React。

## 3. 技术栈选型

| 维度 | 推荐 | 备选 | 理由 |
| --- | --- | --- | --- |
| 构建工具 | **Vite 5** | Next.js / CRA | 启动 < 1s；HMR 极快；原生 ESM |
| 框架 | **React 18** | Vue 3 / Solid | 生态最广；Suspense + concurrent 渲染适合实时 UI |
| 语言 | **TypeScript 5.4+** | – | 类型约束；后端 Pydantic 模式可自动生成 zod schema |
| 状态管理 | **Zustand** | Redux Toolkit / Jotai | 10 行 store；天然支持 selector 订阅，video FPS 场景下减少重渲染 |
| UI 组件库 | **shadcn/ui + Tailwind CSS** | Ant Design / MUI | 按需复制源码，可控样式；与 PyQt 默认观感接近 |
| 图表 | **Recharts** | Visx / ECharts | SVG 渲染友好；与 React 18 并发渲染兼容 |
| 路由 | **React Router 6** | TanStack Router | loader 模式可配合后端预取 |
| 数据请求 | **TanStack Query 5** | SWR / Apollo | 自动缓存 / 失败重试 / 并发请求；模型元数据、评估结果天然 query |
| WebSocket | **native `WebSocket` + reconnect 包装** | socket.io | 视频流是高频二进制帧，避免 socket.io 的 JSON 包装 |
| 表单 | **react-hook-form + zod** | Formik | 性能更好；zod schema 与后端 Pydantic 共享 |
| 视频采集 | **`navigator.mediaDevices.getUserMedia`** | – | 标准 API；摄像头流 |
| 视频推流 | **MediaRecorder + WebSocket binary** | WebCodecs | MediaRecorder 兼容性更好 |
| 测试 | **Vitest + Playwright** | Jest + Cypress | Vitest 与 Vite 同源；Playwright 处理 E2E + 视觉回归 |
| 国际化 | **react-i18next** | – | 中英文切换 |

## 6. 状态管理设计

### 6.1 全局 store（Zustand）

```ts
// src/features/recognition/store.ts
import { create } from "zustand";
import { subscribeWithSelector } from "zustand/middleware";

export interface Prediction {
  class_id: number;
  class_name: string;
  confidence: number | null;
  predict_seconds?: number;
  frame_index?: number;
  reused?: boolean;
}

interface RecognitionState {
  // ── 当前会话 ─────────────────────────────────────
  bundlePath: string | null;
  bundleMeta: { model: string; feature_mode: string; feature_dim: number } | null;
  cacheStats: { hits: number; misses: number; hit_rate: number; size: number };

  // ── 单图模式 ─────────────────────────────────────
  currentImage: ImageBitmap | null;
  preprocessedImage: ImageBitmap | null;
  lastResult: Prediction | null;
  topK: Prediction[];

  // ── 流式模式（video / camera） ──────────────────
  streamMode: "idle" | "video" | "camera";
  streamPaused: boolean;
  lastFrame: ImageBitmap | null;
  fps: number;
  predictMs: number;
  confidenceBuffer: number[];       // EMA 平滑窗口

  // ── 历史 ─────────────────────────────────────────
  history: Array<{ ts: number; src: string; result: Prediction }>;

  // ── Actions ──────────────────────────────────────
  setBundle: (path: string, meta: any) => void;
  setCurrentImage: (img: ImageBitmap, preprocessed: ImageBitmap | null) => void;
  setPrediction: (result: Prediction, topK: Prediction[]) => void;
  setStreamMode: (mode: RecognitionState["streamMode"]) => void;
  pushHistory: (entry: { ts: number; src: string; result: Prediction }) => void;
  clearHistory: () => void;
}

export const useRecognitionStore = create<RecognitionState>()(
  subscribeWithSelector((set) => ({
    bundlePath: null,
    bundleMeta: null,
    cacheStats: { hits: 0, misses: 0, hit_rate: 0, size: 0 },

    currentImage: null,
    preprocessedImage: null,
    lastResult: null,
    topK: [],

    streamMode: "idle",
    streamPaused: false,
    lastFrame: null,
    fps: 0,
    predictMs: 0,
    confidenceBuffer: [],

    history: [],

    setBundle: (path, meta) => set({ bundlePath: path, bundleMeta: meta }),
    setCurrentImage: (img, prep) => set({ currentImage: img, preprocessedImage: prep }),
    setPrediction: (result, topK) =>
      set((s) => ({
        lastResult: result,
        topK,
        history: [
          { ts: Date.now(), src: "single", result },
          ...s.history,
        ].slice(0, 200),
      })),
    setStreamMode: (mode) => set({ streamMode: mode }),
    pushHistory: (entry) =>
      set((s) => ({ history: [entry, ...s.history].slice(0, 200) })),
    clearHistory: () => set({ history: [], confidenceBuffer: [] }),
  }))
);

// 派生选择器：避免每帧重渲染所有订阅者
export const selectFps = (s: RecognitionState) => s.fps;
export const selectLastResult = (s: RecognitionState) => s.lastResult;
```

> 关键：**视频流每秒 25 帧**，如果整个 `<Dashboard>` 组件订阅整个 store，
> 每帧都会重渲染。务必用 selector + `shallow` 比较：
> ```ts
> const fps = useRecognitionStore(selectFps);
> ```

### 6.2 服务端状态（TanStack Query）

模型元数据、评估报告、批量任务进度等用 TanStack Query 缓存：

```ts
// src/features/recognition/api/predict.ts
import { useMutation } from "@tanstack/react-query";

export function usePredict() {
  return useMutation({
    mutationFn: async (input: { bundle: string; image: Blob }) => {
      const form = new FormData();
      form.append("image", input.image);
      const res = await fetch(`/api/predict?bundle=${encodeURIComponent(input.bundle)}`, {
        method: "POST",
        body: form,
      });
      if (!res.ok) throw new Error(`predict failed: ${res.status}`);
      return res.json() as Promise<{
        class_id: number;
        class_name: string;
        confidence: number | null;
        top_k: Array<{ class_id: number; class_name: string; confidence: number }>;
      }>;
    },
  });
}
```

## 7. 后端 API 设计（FastAPI）

新建 `traffic_sign_system/api/` 子包；保留 `Predictor` / `VideoRecognizer` 不变。

```python
# traffic_sign_system/api/app.py
from __future__ import annotations
import io
import time
import uuid
from pathlib import Path
from typing import Annotated

import cv2
import numpy as np
from fastapi import FastAPI, File, Form, HTTPException, Query, UploadFile, WebSocket
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import JSONResponse
from pydantic import BaseModel

from traffic_sign_system.recognition.predictor import Predictor

app = FastAPI(title="Traffic Sign Recognition API", version="1.0.0")
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# ── 全局 registry：bundle_path → Predictor ─────────────
_PREDICTORS: dict[str, Predictor] = {}

def _get(bundle: str) -> Predictor:
    if bundle not in _PREDICTORS:
        try:
            _PREDICTORS[bundle] = Predictor(bundle)
        except Exception as exc:
            raise HTTPException(400, f"failed to load bundle: {exc}")
    return _PREDICTORS[bundle]


# ── 健康检查 ────────────────────────────────────────────
@app.get("/api/health")
def health() -> dict:
    return {"status": "ok", "predictors_loaded": list(_PREDICTORS)}


# ── 单图预测 ────────────────────────────────────────────
@app.post("/api/predict")
async def predict(
    bundle: str = Query(...),
    image: UploadFile = File(...),
    top_k: int = Query(10, ge=1, le=43),
) -> dict:
    predictor = _get(bundle)
    raw = await image.read()
    arr = np.frombuffer(raw, dtype=np.uint8)
    img = cv2.imdecode(arr, cv2.IMREAD_COLOR)
    if img is None:
        raise HTTPException(400, "invalid image bytes")

    t0 = time.perf_counter()
    result = predictor.predict(img)
    predict_seconds = time.perf_counter() - t0

    # Top-K
    top_k_list: list[dict] = []
    if hasattr(predictor.classifier, "predict_proba"):
        feats = predictor.builder.extract_one(img).astype(np.float32, copy=False)
        Xs = predictor.scaler.transform(feats[None, :])
        proba = predictor.classifier.predict_proba(Xs)[0]
        classes = predictor.classifier.classes_
        order = np.argsort(proba)[::-1][:top_k]
        for idx in order:
            top_k_list.append({
                "class_id": int(classes[idx]),
                "class_name": predictor.label_map.get(int(classes[idx]), str(int(classes[idx]))),
                "confidence": float(proba[idx]),
            })

    return {
        **result,
        "predict_seconds": predict_seconds,
        "top_k": top_k_list,
        "cache": predictor.cache_stats(),
    }


# ── 批量预测（返回 task_id，轮询或 SSE） ───────────────
@app.post("/api/batch")
async def batch_predict(bundle: str = Query(...), folder: str = Query(...)) -> dict:
    task_id = str(uuid.uuid4())
    # TODO: 入队（Celery / RQ / asyncio.create_task）
    return {"task_id": task_id, "status": "queued"}


# ── 视频 / 摄像头推流（WebSocket 二进制 + JSON 帧信息） ─
@app.websocket("/ws/stream")
async def stream(ws: WebSocket) -> None:
    await ws.accept()
    cfg = await ws.receive_json()
    bundle = cfg["bundle"]
    skip = int(cfg.get("skip_frames", 2))
    predictor = _get(bundle)

    if cfg.get("src") == "camera":
        cap = cv2.VideoCapture(int(cfg.get("camera_id", 0)))
    else:
        cap = cv2.VideoCapture(cfg["video_path"])

    if not cap.isOpened():
        await ws.send_json({"error": "cannot open stream source"})
        await ws.close()
        return

    frame_index = 0
    last_result: dict | None = None
    t_window = time.perf_counter()
    fps_count = 0
    fps_value = 0.0

    try:
        while True:
            ok, frame = cap.read()
            if not ok:
                await ws.send_json({"done": True})
                break

            frame_index += 1
            do_predict = (frame_index % (skip + 1) == 0) or last_result is None

            t0 = time.perf_counter() if do_predict else 0.0
            if do_predict:
                result = predictor.predict(frame)
                last_result = result
            else:
                result = {**(last_result or {}), "reused": True}
            predict_ms = (time.perf_counter() - t0) * 1000 if do_predict else 0.0

            # 叠加文字
            annotated = frame.copy()
            cv2.putText(annotated, f"{result.get('class_name', '--')}  "
                       f"{result.get('confidence') or 0:.0%}",
                       (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 1.0, (0, 255, 0), 2)

            # 编码 JPEG
            ok, buf = cv2.imencode(".jpg", annotated, [int(cv2.IMWRITE_JPEG_QUALITY), 80])
            if not ok:
                continue

            # 计算 FPS（每秒一次）
            fps_count += 1
            if time.perf_counter() - t_window >= 1.0:
                fps_value = fps_count / (time.perf_counter() - t_window)
                fps_count = 0
                t_window = time.perf_counter()

            await ws.send_bytes(buf.tobytes())
            await ws.send_json({
                "type": "meta",
                "frame_index": frame_index,
                "result": result,
                "predict_ms": predict_ms,
                "fps": fps_value,
                "cache": predictor.cache_stats(),
            })
    finally:
        cap.release()


# ── 模型管理 ────────────────────────────────────────────
@app.get("/api/models")
def list_models() -> dict:
    art = Path("traffic_sign_system/models/artifacts")
    bundles = sorted(p.glob("*.joblib") for p in [art])[0] if art.exists() else []
    return {"bundles": [str(p) for p in art.glob("*.joblib")]}


@app.post("/api/models/{name:path}/load")
def load_model(name: str) -> dict:
    bundle = Path("traffic_sign_system/models/artifacts") / name
    predictor = _get(str(bundle))
    return {
        "model": predictor.classifier.__class__.__name__,
        "feature_mode": predictor.feature_config.get("mode"),
        "feature_dim": predictor.feature_dim,
    }


@app.delete("/api/models/cache")
def clear_cache(bundle: str) -> dict:
    _get(bundle).clear_cache()
    return {"ok": True}
```

### 关键设计点

| 设计 | 价值 |
| --- | --- |
| **WebSocket 二进制帧 + JSON 元数据双通道** | 视频帧用 JPEG 压缩，省带宽；元数据走 JSON 解析快 |
| **Predictor 单例注册表** | 同一 bundle 只 load 一次，复用 `cache_stats` / `_cache` |
| **`/api/batch` 异步任务** | 大文件夹不阻塞；前端用 SSE / 轮询拿进度 |
| **CORS 白名单** | 仅允许前端 dev server 与生产域名 |
| **错误归一** | HTTPException + 自定义异常处理器，前端统一 toast |

### 8.2 视频流（重点）

| 优化 | 实施 | 目标 |
| --- | --- | --- |
| **WebSocket 二进制帧** | JPEG 压缩 80% + 二进制通道 | 带宽 ↓ 70% |
| **前端渲染节流** | `requestAnimationFrame` 绘制 + 跳过过期帧 | 稳定 25-30 FPS |
| **OffscreenCanvas** | 在 Web Worker 绘制叠加层 | 主线程 < 16ms/帧 |
| **缓存复用** | 后端 `Predictor.use_cache=True` | hit_rate > 80% |
| **帧间跳过** | 后端 `skip_frames=2` | FPS 提升 2-3x |
| **批量推理** | 视频帧攒 4 帧一批再发 | 推理吞吐 ↑ |
| **解码分离** | `createImageBitmap` 解码 + 主线程绘制 | 不阻塞 |

示例 WebSocket 客户端（带背压）：

```ts
// src/features/recognition/hooks/useStream.ts
import { useEffect, useRef } from "react";
import { useRecognitionStore } from "../store";

export function useStream(url: string) {
  const setStreamMode = useRecognitionStore((s) => s.setStreamMode);
  const wsRef = useRef<WebSocket | null>(null);
  const canvasRef = useRef<HTMLCanvasElement | null>(null);
  const lastDrawRef = useRef(0);

  useEffect(() => {
    if (!url) return;
    const ws = new WebSocket(url);
    ws.binaryType = "arraybuffer";
    wsRef.current = ws;

    ws.onmessage = async (ev) => {
      // 二进制帧 → 绘制
      if (ev.data instanceof ArrayBuffer) {
        const blob = new Blob([ev.data], { type: "image/jpeg" });
        const bitmap = await createImageBitmap(blob);
        const ctx = canvasRef.current?.getContext("2d");
        // 节流：至少 16ms 才画下一帧
        const now = performance.now();
        if (now - lastDrawRef.current < 16) return;
        lastDrawRef.current = now;
        ctx?.drawImage(bitmap, 0, 0, canvasRef.current!.width, canvasRef.current!.height);
        bitmap.close();
        return;
      }
      // JSON 元数据
      const msg = JSON.parse(ev.data);
      if (msg.result) {
        useRecognitionStore.setState((s) => ({
          lastResult: msg.result,
          fps: msg.fps,
          predictMs: msg.predict_ms,
          cacheStats: msg.cache,
        }));
      }
    };

    return () => ws.close();
  }, [url]);
}
```

### 8.3 计算密集任务

- **图像预处理**（resize / 灰度化 / CLAHE）→ `preprocess.worker.ts`（OffscreenCanvas）
- **bbox 叠加绘制** → `drawOverlay.worker.ts`
- **批量进度解码** → TanStack Query 自带的 `onSettled` 节流

### 8.4 渲染优化

| 场景 | 优化 |
| --- | --- |
| 视频 FPS 数字 | selector 订阅，30Hz 节流到 5Hz |
| 置信度条 | `transform: scaleX()` 替代 `width: %`（GPU 合成） |

## 13. 风险与缓解

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| **WebSocket 跨域代理失败** | 视频流断 | nginx 显式设置 `Upgrade/Connection`；本地用 `vite proxy` |
| **Predictor 内存常驻** | 多模型切换 OOM | 启动时只加载默认 bundle；切换时按需 LRU |
| **JPEG 编码延迟** | 端到端 > 100ms | WebP / H.264 (WebCodecs) 后路方案 |
| **摄像头权限被拒** | 摄像头模式不可用 | 明确错误提示 + 提供"上传视频"备选 |
| **大 bundle 上传** | 网络慢 | chunked upload + 后台进度 |
| **浏览器兼容** | Safari < 16 不支持 OffscreenCanvas | 降级到主线程 + feature detect |

## 总结

把桌面 GUI 搬到 Web 的关键不是"换框架"，而是：

1. **保留后端 Predictor 不动**，只加一层 FastAPI；
2. **WebSocket 二进制推流**让视频端到端延迟可控；
3. **Zustand + 精确 selector**让 30 FPS 视频不会把 UI 拖垮；
4. **Web Worker + OffscreenCanvas**把计算密集任务移出主线程；
5. **代码分割 + 缓存策略**让首屏 < 1s；
6. **Docker Compose + nginx 代理**实现一键部署。

按 5 阶段路线图推进，每阶段交付可独立验证的 demo，整个迁移 6~8 周可上线。
